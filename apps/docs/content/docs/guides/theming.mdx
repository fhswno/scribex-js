title: Theming

Scribex uses CSS custom properties (design tokens) as its theming system. Every token is prefixed with `--scribex-` to prevent collisions with your host application's styles. This approach gives you full visual ownership without forking any components.

## How it works

All Scribex components reference CSS variables instead of hardcoded color values. You control the look and feel by overriding these variables in your own CSS. This is the same pattern used by shadcn/ui.

## Installing the token file

Copy `tokens.css` into your project and import it in your root layout or global stylesheet:

```css title="styles/scribex-tokens.css"
@import './scribex-tokens.css';
```

Or import it directly in your layout:

```tsx title="app/layout.tsx"
import './styles/scribex-tokens.css';
```

## Overriding tokens

Override any token by redefining it in your CSS. Place your overrides after the Scribex token import:

```css title="styles/globals.css"
@import './scribex-tokens.css';

:root {
  /* Use your brand's blue as the accent */
  --scribex-accent: hsl(210 100% 50%);
  --scribex-accent-foreground: hsl(0 0% 100%);

  /* Rounded corners */
  --scribex-radius: 0.75rem;

  /* Custom font */
  --scribex-font-sans: "Inter", ui-sans-serif, system-ui, sans-serif;
}
```

## Dark mode

Scribex supports dark mode via the `.dark` class on a parent element. The token file includes a complete `.dark` block that redefines every token for dark backgrounds:

```css title="tokens.css (dark mode excerpt)"
.dark {
  --scribex-background: hsl(222.2 84% 4.9%);
  --scribex-foreground: hsl(210 40% 98%);
  --scribex-border: hsl(217.2 32.6% 17.5%);
  --scribex-ring: hsl(224.3 76.3% 48%);
  --scribex-muted: hsl(217.2 32.6% 17.5%);
  --scribex-muted-foreground: hsl(215 20.2% 65.1%);
  --scribex-accent: hsl(224.3 76.3% 48%);
  --scribex-accent-foreground: hsl(210 40% 98%);
  --scribex-destructive: hsl(0 62.8% 30.6%);
  /* ... all tokens redefined */
}
```

Toggle dark mode by adding or removing the `.dark` class on your `<html>` or a parent element:

```tsx title="components/ThemeToggle.tsx"
'use client';

import { useEffect, useState } from 'react';

export function ThemeToggle() {
  const [dark, setDark] = useState(false);

  useEffect(() => {
    document.documentElement.classList.toggle('dark', dark);
  }, [dark]);

  return (
    <button onClick={() => setDark((d) => !d)}>
      {dark ? 'Light mode' : 'Dark mode'}
    </button>
  );
}
```

## Tailwind CSS integration

Scribex's Tailwind configuration maps CSS variables to utility classes. If you are extending Tailwind in your own project, you can reference the same variables:

```ts title="tailwind.config.ts"
import type { Config } from 'tailwindcss';

export default {
  theme: {
    extend: {
      colors: {
        'scribex-background': 'var(--scribex-background)',
        'scribex-foreground': 'var(--scribex-foreground)',
        'scribex-border': 'var(--scribex-border)',
        'scribex-ring': 'var(--scribex-ring)',
        'scribex-muted': 'var(--scribex-muted)',
        'scribex-muted-foreground': 'var(--scribex-muted-foreground)',
        'scribex-accent': 'var(--scribex-accent)',
        'scribex-accent-foreground': 'var(--scribex-accent-foreground)',
        'scribex-destructive': 'var(--scribex-destructive)',
      },
      borderRadius: {
        scribex: 'var(--scribex-radius)',
      },
      fontFamily: {
        'scribex-sans': 'var(--scribex-font-sans)',
        'scribex-mono': 'var(--scribex-font-mono)',
        'scribex-serif': 'var(--scribex-font-serif)',
      },
    },
  },
} satisfies Config;
```

This lets you use classes like `bg-scribex-accent`, `text-scribex-foreground`, and `rounded-scribex` in your own components.

## Creating a custom theme

Here is a complete example of a warm, earthy theme:

```css title="styles/earth-theme.css"
:root {
  /* Surface */
  --scribex-background: hsl(30 20% 98%);
  --scribex-foreground: hsl(20 15% 15%);

  /* Chrome */
  --scribex-border: hsl(30 15% 88%);
  --scribex-ring: hsl(25 70% 50%);
  --scribex-muted: hsl(30 15% 94%);
  --scribex-muted-foreground: hsl(20 10% 50%);

  /* Interactive */
  --scribex-accent: hsl(25 70% 50%);
  --scribex-accent-foreground: hsl(0 0% 100%);

  /* Status */
  --scribex-destructive: hsl(0 65% 55%);
  --scribex-ai-stream: hsl(25 70% 50%);

  /* Radius */
  --scribex-radius: 0.375rem;

  /* Typography */
  --scribex-font-sans: "Merriweather Sans", ui-sans-serif, system-ui, sans-serif;
  --scribex-font-mono: "JetBrains Mono", ui-monospace, monospace;
}

.dark {
  --scribex-background: hsl(20 15% 10%);
  --scribex-foreground: hsl(30 20% 92%);
  --scribex-border: hsl(20 12% 20%);
  --scribex-ring: hsl(25 70% 55%);
  --scribex-muted: hsl(20 12% 16%);
  --scribex-muted-foreground: hsl(20 10% 60%);
  --scribex-accent: hsl(25 70% 55%);
  --scribex-accent-foreground: hsl(0 0% 100%);
  --scribex-destructive: hsl(0 50% 40%);
  --scribex-ai-stream: hsl(25 70% 55%);
}
```

Import this file instead of the default `scribex-tokens.css` and every Scribex component will pick up the new theme automatically. No prop changes or component modifications required.

## Important rules

- **Never hardcode colors in components.** Always reference a CSS variable. If a required token does not exist, add it to `tokens.css` first.
- **The `--scribex-` prefix is mandatory.** It prevents collisions with the host app's CSS variables and other component libraries.
- **Dark mode uses the `.dark` class.** Scribex does not use `prefers-color-scheme` by default. You control when dark mode activates by toggling the class.
